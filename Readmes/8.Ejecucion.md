## 🚀 Ejecución de la Aplicación

### Requisitos Previos

Para ejecutar Smart Spend necesitas tener Docker instalado en tu sistema:

#### Windows y Mac
- **Docker Desktop**: [Descargar Docker Desktop](https://www.docker.com/products/docker-desktop/)
- Incluye Docker y Docker Compose automáticamente

#### Linux
- **Docker Engine**: [Instalar Docker en Linux](https://docs.docker.com/engine/install/)
- **Docker Compose**: [Instalar Docker Compose](https://docs.docker.com/compose/install/)

### Ejecución con Docker Compose

1. **Clona el repositorio** (si no lo tienes ya):
```bash
git clone https://github.com/codeurjc-students/2025-SmartSpend.git
cd 2025-SmartSpend
```

2. **Navega al directorio docker**:
```bash
cd docker
```

3. **Ejecuta la aplicación**:
```bash
docker-compose up -d
```

4. **Accede a la aplicación**:
   - Abre tu navegador web
   - Ve a: `https://localhost:443`
   - **Nota**: Como usa certificados autofirmados, acepta el aviso de seguridad del navegador

### Comandos útiles

#### Levantar nueva imágen
```bash
docker build -t juanmabustos/smartspend:dev -f docker/Dockerfile .
```
#### Levantar contenedor 
```bash
docker-compose -f docker-compose-dev.yml up --build
```

#### Detener la aplicación:
```bash
docker-compose -f docker-compose-dev.yml down -v
```

#### Lanzar test de newman*:
```bash
docker-compose -f docker-compose-ci.yml up newman
```



#### Ver logs en tiempo real:
```bash
docker-compose logs -f
```

#### Reiniciar solo la aplicación:
```bash
docker-compose restart app
```

#### Limpiar datos y reiniciar desde cero:
```bash
docker-compose down -v
docker-compose up -d
```

## 🌐 Acceso a la Aplicación en Producción (AWS)

### Versión Desplegada en Producción

### Características del Despliegue en Producción

- **Infraestructura:** AWS EC2 con Docker
- **Base de datos:** MySQL persistente
- **Actualizaciones:** La aplicación se mantiene actualizada con la última versión estable
- **Certificado SSL:** Autofirmado (acepta el aviso en el navegador)

### Diferencia entre Desarrollo y Producción

| Aspecto | Desarrollo Local | Producción (AWS) |
|--------|-----------------|------------------|
| **URL** | `https://localhost:443` | `https://56.228.32.202` |
| **Entorno** | Docker Compose local | EC2 + Docker en AWS |
| **Base de datos** | MySQL en contenedor local | MySQL persistente en EC2 |
| **Scripts de test** | Disponibles (`docker-compose-ci.yml`) | Ejecutados en CI/CD |

### Solución de Problemas

#### La aplicación no carga
- Verifica que Docker esté ejecutándose
- Comprueba que el puerto 8443 no esté ocupado
- Revisa los logs: `docker-compose logs app`

#### Error de certificado SSL
- En tu navegador, acepta el certificado autofirmado
- En Chrome/Edge: Haz clic en "Avanzado" → "Ir a localhost (no seguro)" o "Ir a 56.228.32.202 (no seguro)"
- En Firefox: Haz clic en "Avanzado" → "Aceptar el riesgo y continuar"
- **Nota:** Tanto el entorno local como el de producción usan certificados autofirmados

#### Base de datos no responde
- Espera unos minutos para que MySQL termine de inicializarse
- Verifica el estado: `docker-compose ps`
- Reinicia si es necesario: `docker-compose restart db`
